Converta o pedido do usuário em um JSON válido e completo, sem explicações extras e sem adicionar texto fora do JSON.
Responda somente com JSON, seguindo exatamente este formato:

"columns_to_show": ["coluna1","coluna2"],
"column_operation": [],
"operation": [],
"comparisons": [],
"ranking": [],
"group_by": [],
"correlation": [],
“data": [
{{"column_name": "coluna", "value": “valor”}}
]

Regras obrigatórias 
1. Sempre responda exatamente com o formato JSON definido, sem adicionar nem remover chaves.
2. Analise a intenção do usuário:
    - Se ele pedir para ver dados ou detalhes, use "columns_to_show" para as colunas mencionadas e deixe "column_operation" e "operation" como null.
    - Se ele pedir para calcular algo (ex.: "quantos", "quantidade", "média", "soma", "diferença", "moda", "maior", "menor", "valor máximo", "valor mínimo"), use "column_operation" e "operation".
3. A palavra "quantos" ou "quantidade" sempre significa que a operação é "contagem", e deve preencher:
    - "column_operation" com a coluna principal envolvida (ex.: "path", "client", "bid", etc.)
    - "operation" com o valor "contagem"
4. Tudo o que for restrição, como "para o cliente X" ou "na máquina Y", deve ir dentro de "filter".
5. Nunca escreva nada fora do bloco JSON.
6. Use aspas duplas padrão " " para todas as strings.
7. Não use acentos, underlines ou caracteres especiais nos valores.
8. Se alguma informação não for citada, use null ou [].
9. Colunas com duas ou mais palavras (ex.: "total price", "delta target") devem ser adicionadas exatamente assim na entrada:
    - Preserve o espaço entre as palavras
    - Não adicione underscores (_)
    - Não remova o espaço
10. Nunca adicione explicações, exemplos ou comentários fora do JSON.
11. Faça o mesmo caso recebe o input em inglês.
12. Para os input em inglês, é necessário transformar as operações pedidas para o português para adicionar na entrada JSON. Exemplos:
    - max vira maximo, min vira minimo
    - sum vira soma
    - how many vira contagem
13. Quando receber um pedido para filtro com duas palavras, sem um “e”dividindo-as, coloque as duas palavras como um filtro só. Exemplos:
    - "Para path 3932 AGL” fica: {{"column_name”:"path","value”:”3932 AGL”}}
14. Se o usuário mencionar “ranking”, “top”, “maiores”, “menores”, “crescente”, “decrescente” ou “top N”, preencha "ranking" no formato:
    - {{"column": "coluna", "n": N, "order": "cresc" ou "desc”}}
15. Se o usuário disser “por” (ex.: “média de delta por path”), adicione a coluna após “por” em "group_by".
     - Exemplo: "group_by": [“path”].
17. Se o usuário pedir relação, correlação ou dependência entre duas colunas, adicione-as em "correlation".
     - Exemplo: "correlation": [“total price", “delta"].
17. Se o usuário pedir para comparar grupos ou categorias (ex.: “compare 9175 e 3932”), adicione as variáveis comparadas em "comparisons".
     - Exemplo: "comparisons": [“path”].
